# HookLearn 新手 5 步最短路径

更新时间：2026-04-29

目标：最短时间跑通一条从“设备连通”到“方法追踪与导出”的完整链路。

## 第 1 步：启动 Frida server

参考：doc/1.Frida-server使用.md

执行要点：

1. `adb shell getprop ro.product.cpu.abi`
2. `adb push frida-server /data/local/tmp/`
3. `adb shell chmod 777 /data/local/tmp/frida-server`
4. `adb shell /data/local/tmp/frida-server`

成功标准：

- 能执行 `frida -U -n system_server` 或无连接报错

## 第 2 步：找到目标进程

参考：doc/2.查看所有的进程.md, src/py/ListProcess.py

执行要点：

1. 运行枚举脚本
2. 记录目标进程名/包名

成功标准：

- 获得可 attach 的目标进程

## 第 3 步：跑通注入（优先 Spawn）

参考：doc/4.Spawn模式的使用.md

执行要点：

1. `device.spawn([package_name])`
2. `device.attach(pid)`
3. `script.load()`
4. `device.resume(pid)`

成功标准：

- 看到脚本注入日志且游戏可继续运行

## 第 4 步：跑通 Il2Cpp bridge 与追踪

参考：doc/7.Frida-brige.md, doc/10.追踪方法调用.md, src/brige/Trace.ts

执行要点：

1. 安装依赖：`frida-il2cpp-bridge`, `@types/frida-gum`, `@types/node`
2. 编译 TS 脚本
3. 通过 `Il2Cpp.trace(...).attach()` 追踪目标类

成功标准：

- 触发游戏操作时，终端持续输出目标类方法调用链

## 第 5 步：导出 JSON 与 IDA 重命名脚本

参考：doc/附导出json.md, src/dumpIlp2dumperSo

执行要点：

1. JS 侧批量收集 `namespace/class/method/rva`
2. Python 侧接收 batch 并落盘
3. 同时生成 `ida_rename.py`

成功标准：

- 输出目录中存在 `il2cpp_dump.json` 和 `ida_rename.py`

## 常见阻塞快速排查

1. 设备连不上：检查 USB 调试、adb 设备授权、frida-server 版本与 ABI
2. 进程附加失败：先枚举确认真实进程名，再 attach
3. 脚本无输出：确认 `script.on("message", on_message)` 与 `send(...)` 匹配
4. bridge 不生效：检查加载顺序，先 bridge 后业务脚本
5. 程序崩溃：先关闭高频 Hook 或大批量日志，再逐步打开
