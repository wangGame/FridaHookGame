# HookLearn 统一运行入口说明

更新时间：2026-04-29

目标：把常用 Python/TS/JS 入口统一成一页命令说明，减少路径和启动方式混乱。

## 1. 先决条件

1. 设备已连接并启动 frida-server
2. 主机可用 Python 环境并已安装 `frida`
3. 项目依赖已安装（需要时）

示例（在项目根目录执行）：

```powershell
npm install
```

## 2. 工作目录规则（非常重要）

项目中很多脚本使用相对路径读取 JS 文件，因此运行前要注意当前目录：

1. `src/AppRun.py`、`src/JigsawcardRun.py` 这类脚本，建议在 `src` 目录运行
2. `src/py/*.py` 这类脚本，通常建议在 `src/py` 目录运行

如果目录不对，常见报错是 `No such file or directory`（找不到 JS 路径）。

## 3. 常用入口命令

### 3.1 基础进程与连通

用途：先确认 Frida 通路和目标进程可见。

```powershell
# 在项目根目录
python src/py/ListProcess.py
```

### 3.2 通用 Attach 入口（根 src）

用途：等待或附加目标进程，加载固定脚本。

```powershell
# 建议先进入 src，再运行
Set-Location src
python AppRun.py
python JigsawcardRun.py
```

说明：

1. `AppRun.py` 当前默认加载 `py/xx.js`
2. `JigsawcardRun.py` 当前默认加载 `js/t.js`

### 3.3 Python 场景脚本（src/py）

用途：按场景快速切换 Hook 目标与脚本。

```powershell
Set-Location src/py
python FridaExample.py
python HookUnityGameAddr.py
python HookLog.py
python Dump.py
```

说明：

1. `HookUnityGameAddr.py` 里 `filename` 可切到不同 JS（如 `../dump_il2cpp.js`）
2. `HookLog.py` 当前目标进程写的是 `Block Blast!`，需按实际修改
3. `Dump.py` 用于导出 JSON/生成 IDA 脚本，输出目录需可写

### 3.4 Jigsawcard 场景脚本

用途：Jigsawcard 相关动作与日志追踪。

```powershell
Set-Location src/Jigsawcard
python JigsawcardRun.py
python SendSign.py
```

### 3.5 Cocos 场景脚本

用途：Cocos 游戏的内存与函数信息采集。

```powershell
Set-Location src/cocos
python Dump.py
python CocosPy.py
python CocosPs.py
```

### 3.6 Il2Cpp 导出与 so dump

用途：导出类方法信息、生成辅助分析数据。

```powershell
Set-Location src/dumpIlp2dumperSo
python run_class_dump.py
python Dump_so.py
python dump_CSharp.py
```

### 3.7 libgdx 场景

```powershell
Set-Location src/libgdx
python ArtPuzzle.py
```

## 4. TypeScript 脚本编译入口

用途：把 TS Hook 脚本编译为可注入 JS。

```powershell
# 在项目根目录
npx frida-compile .\src\dump_il2cpp.ts -o .\src\dump_il2cpp.js
```

可按同样方式编译其他 TS 文件，例如：

```powershell
npx frida-compile .\src\Jigsawcard\UnityAnimation.ts -o .\src\Jigsawcard\UnityAnimation.js
```

## 5. 推荐执行顺序（命令视角）

1. `python src/py/ListProcess.py`
2. `Set-Location src; python AppRun.py`
3. `npx frida-compile .\src\dump_il2cpp.ts -o .\src\dump_il2cpp.js`
4. `Set-Location src/py; python HookUnityGameAddr.py`
5. `Set-Location src/py; python Dump.py`

## 6. 常见错误与处理

1. `ProcessNotFoundError`：先跑 `ListProcess.py` 确认进程名
2. `No such file or directory`：修正当前工作目录再运行脚本
3. `Il2Cpp is not defined`：先加载 bridge，再加载业务脚本
4. 无任何日志：检查 `script.on("message", on_message)` 与 JS 端 `send(...)`
